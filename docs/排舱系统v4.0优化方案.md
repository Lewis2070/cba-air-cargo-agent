## 一、机型专业性整改：B767-300F

### 1.1 为什么要专注 B767-300F？

B767-300F（Foldable Main Deck Freighter）是目前全球货运的主流机型之一，具有：
- 最大有效载荷：约 52,000 kg（115,000 lbs）
- 主舱门尺寸：可装载 PMC、PAG、PLA 等大型 ULD
- 下货舱：适合 AKE、RKN 等小型集装箱
- 运营成本效益高，是 DHL、FedEx、UPS 等头部货运航空的核心机型

## 二、载重平衡包线图（Weight & Balance Envelope）业务逻辑

### 2.1 核心定义

**包线图**是确保飞行安全的核心工具，由纵轴（总重量）和横轴（重心位置 %MAC）构成的封闭多边形。

```
标准 B767-300ERG 包线示意图（参考）：

重量(lbs)
345,000 ────────[================MTOW limit================]──→ (最大起飞重量)
                ╱                                            ╲
               ╱  ← FORWARD CG LIMIT                        ╲
              ╱                                              ╲
              ╲                                              ╱
295,000 ────────[===============MZFW limit==================]──→ (最大零油重量)
              ╲    ↑                                        ╱
               ╲  ← AFT CG LIMIT                            ╱
                ╲                                            ╱
230,000 ─────────[==========MAX LANDING WEIGHT============]──→
              ╲
               ╲ ← STRUCTURAL LIMITS (后重心限制)
                ╲
                 ╲
130,000 ──────────[=======================================]──→ (空机+燃油最小重量)
                  ───────────────────────────────────────────────→
                14%      20%     28%      35%      42% (%MAC)
                ↑       ↑       ↑       ↑       ↑
             前限制   巡航   起飞    落地    后限制
              极限   极限   极限   极限     极限
```

### 2.2 六条关键限制线

| # | 限制名称 | 定义 | 公式/典型值 |
|---|---------|------|-----------|
| 1 | **MTOW** (最大起飞重量) | 飞机结构 + 动力允许的最大起飞总重量 | 典型值：345,000 lbs（156.4t） |
| 2 | **MZFW** (最大零燃油重量) | 无燃油时的最大结构允许重量 | 典型值：295,000 lbs（133.8t） |
| 3 | **MLW** (最大落地重量) | 飞机结构允许的最大落地总重量 | 典型值：285,000 lbs（129.3t） |
| 4 | **前重心限制** (Forward CG Limit) | 保证足够俯仰配平能力 | 随重量变化：轻载时 ~14% MAC，重载时 ~25% MAC |
| 5 | **后重心限制** (Aft CG Limit) | 保证飞行稳定性 | 随重量变化：轻载时 ~42% MAC，重载时 ~35% MAC |
| 6 | **空机重量线** (空机+最小业务载重) | 飞机最低起飞重量 | 典型值：~130,000 lbs |

### 2.3 三阶段重心判断逻辑

```
重心指数计算公式：
CG(%) = [(实际重心力臂 - MAC_LE) / MAC] × 100

三阶段判断（Takeoff / Cruise / Landing）：

Takeoff 阶段（起飞）：
  - 燃油重量在最大起飞燃油位
  - 重心受 MTOW + 前/后重心限制约束
  - 安全范围：Forward Limit ~ 28% MAC 至 Aft Limit ~ 35% MAC

Cruise 阶段（巡航）：
  - 燃油消耗后重心前移（燃油重心通常靠前）
  - 可以接受更靠后的起飞重心（因为燃油消耗会使重心前移）
  - 安全范围：Forward Limit ~ 20% MAC 至 Aft Limit ~ 42% MAC（最宽松）

Landing 阶段（落地）：
  - 燃油已消耗，重量降至 MLW
  - 重心已在起飞时前移，但需要检查后限
  - 安全范围：Forward Limit ~ 20% MAC 至 Aft Limit ~ 38% MAC
```

### 2.4 载重变化对重心影响的物理原理

```
燃油消耗 → 重心前移（燃油箱通常位于机翼重心偏前位置）
货物卸载 → 重心根据卸载位置决定前后移动
加油 → 重心根据燃油位置决定前后移动

平衡方程：
Σ(重量_i × 力臂_i) = 总力矩
CG(力臂) = 总力矩 / 总重量
CG(%) = (CG(力臂) - MAC_LE) / MAC × 100
```

## 三、当前系统问题分析

| # | 问题 | 严重性 | 根因 |
|---|------|--------|------|
| 1 | 显示 B747/B777 等错误机型数据 | 🔴 高 | 当前系统内置了747/777数据，但用户确认为错误 |
| 2 | 舱位布局不符合 B767-300F 实际配置 | 🔴 高 | 舱位数量、位置、arm值均未对标767实机数据 |
| 3 | 包线图错误（仅显示简单进度条） | 🔴 高 | 当前用 Progress bar 代替了真实包线多边形 |
| 4 | 没有 MZFW / MLW / MTOW 限制线 | 🔴 高 | 包线图缺少六条关键限制 |
| 5 | 侧边主菜单不支持折叠 | 🟡 中 | 没有折叠/展开功能，作业区太小 |
| 6 | 没有深色/浅色主题切换 | 🟡 低 | 业务需要夜间作业支持 |

### Phase 2：载重平衡包线图（2天）

**目标：** 用 SVG 实现专业包线图，替代现有的简单进度条

**任务清单：**
- [ ] 新建 `<WeightBalanceEnvelope />` 组件（SVG 实现）
- [ ] 绘制六条限制线（MTOW、MZFW、MLW、Forward CG、Aft CG、空机线）
- [ ] 绘制"当前重心点"随载重变化的移动轨迹（从起飞→巡航→落地的实时跟踪）
- [ ] 实现包线内"当前重心点"的动态显示（随货物装载实时移动）
- [ ] 重心越限时的告警高亮（红色危险区着色）

**SVG 包线图数据结构：**
```typescript
interface EnvelopePoint {
  weight: number;   // 重量 (lbs 或 kg)
  cgPct: number;   // 重心位置 (%MAC)
}

interface LimitLine {
  label: string;
  points: EnvelopePoint[];  // 多边形顶点
  color: string;
  fillColor?: string;       // 限重区填充色
}

interface EnvelopeChartProps {
  currentWeight: number;     // 当前总重量
  currentCGPct: number;      // 当前重心 %
  limits: {
    mtow: LimitLine;
    mlw: LimitLine;
    mzfw: LimitLine;
    forwardCG: LimitLine;
    aftCG: LimitLine;
    emptyWeight: LimitLine;
  };
  phase: 'takeoff' | 'cruise' | 'landing';
}
```
