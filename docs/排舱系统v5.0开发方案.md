### 1.3 重心包线参数（B767-300BCF，参考 FCOM）

```
Y轴（重量）：0 ~ 412,000 lbs
X轴（CG）：0% ~ 50% MAC

六条包线限制（依据波音 FCOM B767-300）：

1. MTOW 水平线：y = 412,000 lbs（顶部硬限制）
2. MZFW 水平线：y = 329,000 lbs（结构限重）
3. MLW 水平线：y = 367,000 lbs（落地限重）
4. 前重心限制 Forward CG（倾斜边，随重量变化）：
   - 重量 189,000 lbs → 前限 14% MAC
   - 重量 329,000 lbs → 前限 22% MAC
   公式近似：CG_fwd = 0.03 × (W - 189000) / 140000 + 0.14
5. 后重心限制 Aft CG（倾斜边，随重量变化）：
   - 重量 189,000 lbs → 后限 43% MAC
   - 重量 329,000 lbs → 后限 35% MAC
   公式近似：CG_aft = -0.058 × (W - 189000) / 140000 + 0.43
6. 空机重量点：约 189,200 lbs @ 20% MAC（参考）
```

## 二、载重平衡包线图（Weight & Balance Envelope）设计规范

### 2.1 SVG 包线图技术要求

```typescript
// 包线图数据结构
interface W&BEnvelope {
  // 六条边界线
  mtow:    { x: number[]; y: number[] }; // 412,000 lbs 水平线
  mzfw:    { x: number[]; y: number[] }; // 329,000 lbs 水平线
  mlw:     { x: number[]; y: number[] }; // 367,000 lbs 水平线
  fwdCG:   { x: number[]; y: number[] }; // 前重心限制（斜线）
  aftCG:   { x: number[]; y: number[] }; // 后重心限制（斜线）
  emptyPt: { x: number; y: number };      // 空机重量点

  // 三阶段标注
  takeoff:  { x: number[]; y: number[] };  // 起飞阶段（最严格）
  cruise:   { x: number[]; y: number[] };  // 巡航阶段（允许更宽后重心）
  landing:  { x: number[]; y: number[] };  // 落地阶段
}

// 重心轨迹颜色
const PHASE_COLORS = {
  takeoff:  '#EF4444',  // 红色
  cruise:   '#F59E0B',  // 橙色
  landing:  '#10B981',  // 绿色
};
```

### 2.2 包线图布局规格

```
SVG 尺寸：width=600, height=480
Y轴范围：0 ~ 420,000 lbs（留余量）
X轴范围：0% ~ 50% MAC（留余量）

五边形的五个顶点（由前限+后限+MTOW+MZFW+MTOW交点构成）：

点1（MTOW前限）：MTOW线与Forward CG线的交点
点2（MTOW后限）：MTOW线与Aft CG线的交点
点3（MZFW后限）：MZFW线与Aft CG线的交点
点4（MZFW前限）：MZFW线与Forward CG线的交点
点5（前限下端）：Forward CG线与空机重量线的交点

安全包络区域（绿色/蓝色填充）：上述五边形内部
当前重心点：大号红色实心圆，直径12px，实时移动
重心轨迹：虚线连接起飞→巡航→落地的历史轨迹
越限区域：红色高亮（CG超出前限或后限）
```

### 2.3 重心计算公式

```
CG(力臂) = Σ(重量_i × 力臂_i) / Σ重量_i
CG(%MAC) = (CG(力臂) - MAC_LE) / MAC × 100

三阶段重心越限判断：
  if (CG_pct < Forward_Limit)  → 🚨 前重心越限（起飞拉起不足）
  if (CG_pct > Aft_Limit)     → 🚨 后重心越限（稳定性不足）
  if (TotalWeight > MTOW)      → 🚨 超起飞重量
  if (TotalWeight > MZFW && fuel > 0) → 🚨 零燃油超重
```

### 1.6 装载合理性三维判断算法

**体积填充率算法（核心）：**

```
cargo_vol_i = 长(cm) × 宽(cm) × 高(cm) / 6000  → 体积重(kg)
chargable_vol = max(cargo_vol_i, actual_weight_kg)  → 计费体积

ULD剩余体积 = ULD_internal_vol - Σ(已装货物体积)
ULD剩余载重 = ULD_max_weight - Σ(已装货物重量) - ULD_tare_weight

if (cargo_vol > ULD剩余体积) → ⚠️ 货物装不下，该ULD不可选
if (cargo_weight > ULD剩余载重) → ⚠️ 超重，该ULD不可选
if (cargo.is_dgr && 已有非DGR货物) → ⚠️ DGR隔离规则冲突
```

**合理性评级：**

| 填充率 | 评级 | 说明 |
|--------|------|------|
| 100% | 🟢 完美 | 体积刚好填满 |
| 80~99% | 🟢 优秀 | 合理利用空间 |
| 60~79% | 🟡 一般 | 有浪费但可接受 |
| 30~59% | 🔴 浪费 | 建议换更大ULD |
| < 30% | ⚫ 严重浪费 | 必须换ULD |
| > 100% | 🚨 超载 | 物理装不下 |

## 六、ULD 3D 内部可视化技术方案

### 6.1 三维建模（CSS 3D Transforms）

每个 ULD 渲染为带深度感的3D箱子，货物堆叠其中：

```tsx
// ULD 3D箱子
<div style={{ perspective: 800, transformStyle: 'preserve-3d' }}>
  {/* 前脸（舱门侧） */}
  <div className="uld-front">{ULD_ID}</div>

  {/* 货物块（堆叠，按y轴分层） */}
  {cargoItems.map((item, i) => (
    <div
      key={item.id}
      className="cargo-block"
      style={{
        width: `${(item.length_cm / uld.length_cm) * 100}%`,
        height: `${(item.height_cm / uld.height_cm) * 100}%`,
        depth: `${(item.width_cm / uld.width_cm) * 100}%`,
        translateZ: `${accumulatedHeight}px`,
        background: getCargoColor(item.category),
      }}
    />
  ))}

  {/* 剩余空间指示 */}
  {remainingPct > 0 && (
    <div className="empty-space" style={{ opacity: remainingPct / 100 }} />
  )}
</div>
```

### 6.3 ULD选择推荐算法

用户输入AWB体积后，系统自动推荐最优ULD：

```typescript
function recommendULD(volume_m3: number, weight_kg: number): ULDType[] {
  const candidates = ULD_TYPES.filter(uld =>
    uld.maxVolume_m3 >= volume_m3 &&
    uld.maxWeight_kg >= weight_kg
  );
  // 按填充率最优排序
  return candidates.sort((a, b) =>
    Math.abs(a.volume_m3 - volume_m3) - Math.abs(b.volume_m3 - volume_m3)
  );
}
```
