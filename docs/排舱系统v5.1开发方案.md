## 二、行业标准研究

### 2.1 B767-300BCF 货舱配置（依据：波音 FCOM + Atlas Air One Sheet）

```
总货舱容积：445.3 m³
最大有效载荷：56,500 kg
航程：3,345 海里（6,190 km）

主舱（Main Deck）：
  - 24 个货盘位（部分改装为22个）
  - 容积：336.5 m³
  - 代码：M1, M2, M3... M22（或按列/行分组）
  - 适用：Q7(LD-7), Q6(LD-6), Q6(LD-6) 板型

下舱（Lower Deck）：
  - 前货舱 L1 ~ L6（约60m³）
  - 后货舱 L7 ~ L12（约36m³）
  - 总容积：96.6 m³
  - 适用：AKE(LD-3), AAU(LD-2) 板型

鼻舱（Nose Hold / Forward Cargo）：
  - N1, N2（容积较小，约 6~8m³）
  - 适用：小件/紧急货物
```

### 2.3 IATA DGR 危险品隔离规则（2025版）

**各类危险品隔离组别及隔离距离要求：**

```
类别        子类          隔离要求                    典型UN编号
-----------------------------------------------------------------------------
爆炸品      1.1, 1.2      完全隔离（不同舱区）        UN0081, UN0082
易燃气体    2.1           与易燃液体隔离              UN1049, UN1950
易燃液体    3             与氧化剂/爆炸品隔离         UN1993, UN1170
氧化剂      5.1           与可燃物/易燃液体隔离        UN1072, UN1942
有机过氧化物 5.2           与所有其他货物隔离          UN3101, UN3108
毒性/感染物 6             与食品隔离                  UN2810, UN2900
放射性      7             与人员/易燃物隔离            UN2982
腐蚀性      8             与易燃物/硝酸隔离            UN1789, UN2796
```

**隔离规则：**
- 同一飞行器内，隔板分离（Different compartments）或对角线位置
- 主舱与下舱通常视为隔离（由金属舱壁分隔）
- 鼻舱与主舱之间可能有空气循环，查询具体适航要求

### 2.4 IATA 活体动物/生鲜货物规定（LAPB / LAR / PCR）

```
活体动物（LAR）：
  - 必须与危险品、普通货物隔离
  - 最少一个完整舱区（如整个下舱前段）
  - 必须通风（不能放在无通风的死角）
  - 温度范围：特定物种有要求（冷血/温血/爬行类不同）
  - 必须申报UN编号及包装

生鲜货物（PCR）：
  - 必须与危险品隔离
  - 温度控制：冷藏（2-8°C）/冷冻（-18°C以下）
  - 不能与产生异味的货物同舱（如海鲜/肉类）
  - 部分国家有清关要求（植检/兽医证明）
```

### 2.5 重心（CG）包线图标准（B767-300BCF）

```
X轴：% MAC（平均气动弦长百分比）
Y轴：实际起飞重量（kg）

限制曲线（各飞行阶段）：
  - 起飞（TO）：9% ~ 33% MAC，最高 186,880 kg（MTOW）
  - 巡航（CR）：11% ~ 38% MAC
  - 落地（LD）：9% ~ 38% MAC，最高 170,500 kg（MLW）

实用重心范围（运营建议）：
  - 前限：9% MAC（起飞构型）
  - 后限：33% MAC（起飞）/ 38%（巡航）

静止重心（空载）：
  - 空机重量重心约 15-18% MAC
  - 全空时 CG 约 25% MAC（非偏前）
  - 用户反馈"0.0%"说明计算逻辑有误（应为空机重心，而非0%）
```

**ULD 智能推荐算法：**
```typescript
function recommendULD(cargo: CargoItem): string[] {
  const volWeight = cargo.weight_kg; // 体积重（IATA /6000）
  const chargeable = cargo.chargeableWeight_kg;
  // 按载重匹配 + 填充率最优排序
  return ULD_TYPES
    .filter(u => u.max_load_kg >= chargeable)
    .sort((a,b) => {
      const fillA = cargo.volume_m3 / a.volume_m3;
      const fillB = cargo.volume_m3 / b.volume_m3;
      // 优先填充率在 50-90% 之间的
      const scoreA = fillA >= 0.5 && fillA <= 0.9 ? 100 - Math.abs(fillA-0.7)*100 : 0;
      const scoreB = fillB >= 0.5 && fillB <= 0.9 ? 100 - Math.abs(fillB-0.7)*100 : 0;
      return scoreB - scoreA;
    })
    .slice(0, 3)
    .map(u => u.code);
}
```

```

  M1前段: -5%    M1: -10%    M2: -20%    M3: -30%
  M4: -40%      M5: -50%    M6: -60%
  N1/N2: -10%（鼻舱）

  L5: +50%      L6: +60%    L7: +65%   L8: +75%
  L9: +80%      L10: +85%   L11: +90%  L12: +95%
```

### 4.6 AI 一键排舱算法

**目标函数：**
```
最大化：Σ(货物_i.计费重_kg × 货物_i.费率_元/kg)
约束条件：
  1. Σ重量 ≤ 52,000 kg（实际可用载重）
  2. CG (%) 必须在 TO 限制曲线内（9%~33%）
  3. 同 ULD 内：DGR 隔离规则
  4. 同舱区内：活体/生鲜 隔离要求
  5. 每个舱位限重
```

**算法步骤：**
```
1. 贪心初始化：按计费重/体积比降序排列货物
2. 优先装计费重高的货物（利润最大化）
3. 每步计算 CG，若超出则换舱或换ULD
4. DGR 冲突触发时，强制分ULD或提示人工干预
5. 活体货物固定分到下舱L1/L2（通风最好）
6. 生鲜货物分配到温控区（避免与DGR混合）
7. 产出多方案供选择
```
